# 比亚迪汉LEV里程计数器

这是一个个人自用的离线 PWA 网页小程序，用来记录“日期 + 车机显示总里程”，并估算任意连续 12 个月内的累计行驶里程是否接近 30000 km 上限。

数据只保存在当前手机或浏览器本地，不需要登录，不上传数据，不调用 AI，也不需要后端服务器。

## 规则说明

厂家规则按“任意连续 12 个月滑动窗口”理解：例如 2026-07-08 要往前看 12 个月到 2025-07-08，计算这段窗口内的行驶里程。

这不是自然年度重置，不是每年 1 月 1 日清零，也不是从购车日起每 12 个月固定重置。

## 在电脑上打开

最推荐用本地小服务器打开，因为 PWA 离线缓存需要 `localhost` 或 HTTPS。

1. 打开命令行，进入本目录。
2. 运行：

```bash
python -m http.server 4173
```

3. 在浏览器打开：

```text
http://localhost:4173
```

也可以直接双击 `index.html` 查看和记录数据，但这种方式通常不能注册 PWA 离线缓存。

## 在手机上使用

推荐把手机作为主设备，因为数据默认保存在当前设备本地。

简单方式：

1. 把整个 `byd-mileage-tracker` 文件夹上传到 GitHub Pages 或其他静态网页空间。
2. 用华为手机浏览器打开网页地址。
3. 第一次打开时，选择“手动建立车辆档案”或“导入已有数据”。
4. 后续每次记录当天车机显示的总里程即可。

局域网临时试用：

1. 电脑和手机连接同一个 Wi-Fi。
2. 电脑在项目目录运行 `python -m http.server 4173`。
3. 查询电脑局域网 IP，例如 `192.168.1.23`。
4. 手机浏览器打开 `http://192.168.1.23:4173`。

## 添加到华为手机桌面

1. 用华为浏览器或 Chrome 打开网页。
2. 点浏览器菜单。
3. 选择“添加至桌面”“添加到主屏幕”或“安装应用”。
4. 桌面出现图标后，以后可以像普通 App 一样打开。

如果菜单里没有安装选项，请优先使用 GitHub Pages 的 HTTPS 地址，或使用 `localhost` 调试；直接从文件管理器打开 HTML 往往不能安装 PWA。

## 备份 JSON

在页面下方“数据管理”区域点击“导出 JSON 备份”。这个文件包含车辆档案和全部里程记录，适合换手机、换浏览器、恢复数据。

建议：初始化完成后立刻导出一次 JSON，以后每隔一段时间再备份一次。

## 导入 JSON 恢复

在第一次打开时可以点“导入已有数据”。已经有数据时，也可以在“数据管理”区域点击“导入 JSON 恢复”。

导入时会校验：

- 是否包含 `vehicleProfile`
- 是否包含 `records`
- 日期格式是否正确
- 总里程是否非负
- 按日期排序后，总里程是否非递减
- 是否有重复日期
- 是否存在超过 2000 km 的异常大幅增长

如果 JSON 文件内同一天有重复记录，第一版会保留 `updatedAt` 较新的那条。

## 导出 Markdown 给 GPT

在“数据管理”区域点击“导出 Markdown”。生成的 `.md` 文件包含车辆档案、当前概况和表格化里程记录，可以直接复制给 GPT 或自己归档分析。

## 导出 CSV 给 Excel

点击“导出 CSV”会生成 Excel 可打开的记录表，字段为：

```text
date, odometerKm, note, createdAt, updatedAt
```

## 部署到 GitHub Pages

1. 新建一个 GitHub 仓库。
2. 把 `byd-mileage-tracker` 目录里的文件上传到仓库根目录。
3. 进入仓库 `Settings`。
4. 打开 `Pages`。
5. Source 选择 `Deploy from a branch`。
6. Branch 选择 `main` 和 `/root`。
7. 保存后等待 GitHub 给出 Pages 地址。
8. 用手机打开这个 HTTPS 地址，然后添加到桌面。

如果你想保留仓库里还有其他文件，也可以把本目录放在子目录中，但要确认访问地址能打开 `index.html`。

## 为什么第一版不做云同步

第一版优先保证简单、离线、隐私和可维护。

- 数据保存在当前设备本地。
- 手机和电脑打开同一个 PWA 地址时，数据不会自动同步。
- 推荐手机作为主设备。
- 电脑端只导入查看，尽量不要同时编辑。
- 需要迁移或恢复时，用 JSON 导出和导入。

代码已经为未来同步预留了字段：每条记录都有 `id`、`createdAt`、`updatedAt`。

## 数据隐私

- 数据保存在本机浏览器 `localStorage`。
- 不上传到服务器。
- 不调用 AI 接口。
- 不需要登录账号。
- 不依赖 CDN 或外部库。

请注意：清理浏览器缓存、卸载浏览器或更换手机，都可能导致本地数据丢失，所以要定期导出 JSON 备份。

## 手动测试清单

1. 第一次打开，应显示初始化向导。
2. 选择“手动建立车辆档案”。
3. 使用购车时间 `2025年7月左右`，首次记录 `2026-07-08`，总里程 `27413`。
4. 首页应显示剩余约 `2587 km`，计算方式为“保守估算”。
5. 新增后一条记录，确认首页和历史记录立即刷新。
6. 输入小于前一条的总里程，应禁止保存并说明原因。
7. 同日期再次保存，应弹出覆盖提示。
8. 增加超过 `2000 km`，应弹出确认提示。
9. 导出 JSON，确认生成备份文件。
10. 刷新页面，确认数据仍存在。
11. 删除记录，确认删除前有提示，删除后重新计算。
12. 导入 JSON，确认恢复车辆档案和记录。
13. 导出 Markdown，确认表格适合复制给 GPT。
14. 修改安全余量，确认建议可用额度同步变化。
15. 修改上限、预警线或高风险线，确认风险等级同步变化。
16. 添加超过 12 个月跨度的记录，确认计算不是按自然年度清零。
17. 检查三块趋势区域能正常显示。

## 常见问题

### 换手机怎么办？

旧手机先导出 JSON 备份，把文件发到新手机。新手机打开网页后，选择“导入已有数据”。

### 清缓存数据没了怎么办？

如果之前导出过 JSON，重新导入即可。如果没有备份，浏览器本地数据通常无法恢复。

### 为什么显示“保守估算”？

因为缺少 12 个月前附近的真实里程记录。程序会用购车时里程做保守参考，不会假装精确。随着你持续记录，插值估算或精确计算会逐渐出现。

### 为什么不是每年清零？

因为规则是任意连续 12 个月滑动窗口。每一条记录都要往前看 12 个月，而不是按自然年度计算。

### 购车日期不精确怎么办？

可以填写 `2025-07` 或 `2025年7月左右`。程序会保存原始文本，并用一个估算日期辅助早期保守计算。
